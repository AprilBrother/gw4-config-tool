Testing guide

Testing mode can be used to verify whether the bluetooth gateway uploads data properly. Normally, the host above should be set to the ip address of a server. In the testing mode, this computer (on which this configuration tool is running) acts as the server to receive data from the AB gateway
To set up testing,
Set Connection Type to HTTP client
Set Host to the ip address of this computer
Set Port to 50080
Set URI to /app.php
Enable in-bound data transfer on port 50080 of this computer in the firewall configuration.
Click Save and Restart

测试指南

测试功能是检查此台网关是否正常上传数据的功能。一般情况下上面的服务器Host框应当填写服务器地址。在此测试功能下，当前配置网关的这台电脑就临时充当服务器进行接收数据跟解码工作。
在测试状态下的配置是：
host为当前电脑的ip地址。
Connection Type为 HTTP Client
Port为50080
URI为 /app.php
在本机放火墙设置里打开端口50080
点击Save后此台网关将向本机发送数据。

Introduction

This is a tool for configuring the AB Bluetooth Gateway, it runs on a windows PC (config machine). The AB Bluetooth Gateway can be accessed in two modes, AP mode and STA mode.
Out of box, only AP mode is turned on.
To access in STA mode, first one uses a config machine to connect the AB Bluetooth Gateway in AP mode, then configure that AB Bluetooth Gateway to connect to some wifi AP (e.g. common_AP), then switch the config machine to connect to the same wifi AP which the AB Bluetooth Gateway is connecting. Access in STA mode

When accessed in STA mode, the AB Bluetooth Gateway acts as a wifi client and connects to a specified nearby wifi AP, the config machine acts also as a wifi client and connects to the same wifi AP as the AB Bluetooth Gateway connects.

To access in STA mode,
first one uses a config machine to connect the AB Bluetooth Gateway in AP mode
then configure that AB Bluetooth Gateway to connect to some wifi AP (e.g. common_AP)
then switch the config machine to connect to the same wifi AP which the AB Bluetooth Gateway is connecting. (e.g. common_AP)
Click in the left pane will list all AB Bluetooth Gateways connected to the same wifi AP as the config machine is connecting.

STA模式接入

所有的网关交付客户后首先客户需要进行AP模式的配置。当配置好此网关的上网Wifi(输入SSID和密码)以后。此网关可以连接局域网的Wifi。这样以后就可以不再使用AP模式对它进行配置。可以使用STA模式配置。 此时左侧列表显示的都是可以使用STA模式进行配置的当前局域网里的网关.
